[PATCH-01] إصلاح SSH MITM Vulnerability (حرج)
═══════════════════════════════════════════════
الملف: pc-commander/src/pc_control/ssh_executor.py
المشكلة: AutoAddPolicy() يقبل أي host key بدون تحقق
الإصلاح:
- استبدل AutoAddPolicy بـ RejectPolicy
- أضف منطق لتحميل known_host_key من config أو ملف منفصل
- إذا لم يوجد known_host_key، اطلب من المستخدم تأكيد البصمة عند الاتصال الأول وخزّنها (Trust On First Use - TOFU)
- أضف validation للـ fingerprint قبل كل اتصال

[PATCH-02] إزالة shell=True وتطبيق Input Sanitization (حرج)
════════════════════════════════════════
الملف: pc-commander/src/pc_control/process_manager.py
المشكلة: shell=True + os.system() = command injection
الإصلاح:
- في دالة run_command(): استبدل shell=True بـ shlex.split() لتحويل الأمر لقائمة args:
  import shlex
  result = subprocess.run(shlex.split(command), capture_output=True, text=True, timeout=30, encoding="utf-8", errors="replace")
- في shutdown_pc() وrestart_pc() وcancel_shutdown(): استبدل os.system() بـ subprocess.run مع args كقائمة:
  subprocess.run(["shutdown", "/s", "/t", str(delay_minutes * 60)], check=False)
- أضف قائمة ALLOWED_COMMANDS: set = {"shutdown", "restart", "screenshot", "system_status", ...} وتحقق منها قبل أي تنفيذ

[PATCH-04] حماية file_manager من Path Traversal (عالية)
═══════════════════════════════════════════════
الملف: pc-commander/src/pc_control/file_manager.py
الإصلاح:
- أضف ALLOWED_BASE_PATHS في config: قائمة بالمسارات المسموح بالوصول إليها (مثل: home directory، Desktop، Documents)
- أنشئ دالة is_safe_path(path: str) -> bool:
  def is_safe_path(path: str) -> bool:
      resolved = Path(path).resolve()
      return any(
          resolved == base or base in resolved.parents
          for base in [Path(p).resolve() for p in ALLOWED_BASE_PATHS]
      )
- طبّقها على: list_directory, delete_file, copy_file, move_file, open_file, get_file_info, search_files
- ارفض أي مسار خارج القائمة المسموح بها مع رسالة واضحة
