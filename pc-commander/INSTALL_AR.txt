1. نبذة عن البرنامج
═══════════════════════════════════

NexAgent هو نظام متكامل للتحكم الكامل بحاسبك عن بُعد، يعتمد على طبقتين:

  الطبقة الأولى — بوت تيليغرام الذكي:
    يتيح لك التحكم بالحاسب عبر أوامر طبيعية بالعربية أو الإنجليزية.
    يستخدم GPT-4o أو Gemini 1.5 Flash لفهم الأوامر وتنفيذها.

  الطبقة الثانية — نفق SSH (SshRemote V2):
    وصول SSH آمن بدون كلمة مرور عبر نفق bore تلقائي.
    يعمل بدون IP ثابت أو فتح منافذ في الراوتر.
    يُرسل منفذ الاتصال إلى تيليغرام تلقائياً عند كل اتصال.

الإمكانيات الرئيسية:
  • لقطة شاشة فورية
  • فتح/إغلاق التطبيقات
  • إدارة الملفات
  • مراقبة CPU / RAM / درجة الحرارة
  • التحكم البصري الذكي (يرى الشاشة وينقر)
  • إيقاظ الحاسب عن بُعد (Wake-on-LAN)
  • بث الشاشة مباشرة
  • تقارير يومية تلقائية
  • تنفيذ أوامر shell عبر SSH
  • رفع/تنزيل الملفات عبر SFTP

═══════════════════════════════════
2. متطلبات التشغيل
═══════════════════════════════════

  - ويندوز 10 أو 11 (64-bit)
  - Python 3.11 أو أحدث  <  python.org/downloads  >
    (أثناء التثبيت: فعّل "Add Python to PATH")
  - اتصال بالإنترنت
  - مفتاح API: OpenAI (gpt-4o) أو Google Gemini
  - للطبقة الثانية: صلاحيات مدير (Administrator) لتثبيت OpenSSH

═══════════════════════════════════
3. إنشاء بوت تيليغرام
═══════════════════════════════════

  أ) افتح تيليغرام وابحث عن @BotFather
  ب) أرسل الأمر:  /newbot
  ج) اختر اسماً للبوت (مثال: My NexAgent Bot)
  د) اختر اسم مستخدم ينتهي بـ "bot" (مثال: my_nexagent_bot)
  هـ) احفظ رمز "HTTP API token" الذي سيُعطيك إياه BotFather
      مثال: 7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

═══════════════════════════════════
4. معرفة Chat ID الخاص بك
═══════════════════════════════════

  أ) افتح تيليغرام وابحث عن @userinfobot
  ب) أرسل أي رسالة له
  ج) سيردّ بمعلوماتك — احفظ الـ "Id" (رقم مثل: 123456789)

═══════════════════════════════════
5. تثبيت الطبقة الأولى — بوت AI
═══════════════════════════════════

الطريقة الأولى — التثبيت التلقائي (مُوصى به):
  أ) انقر بزر الأيمن على install.bat
  ب) اختر "تشغيل كمسؤول"
  ج) اتبع التعليمات على الشاشة

الطريقة الثانية — التثبيت اليدوي:
  أ) تأكد من تثبيت Python 3.11+
  ب) افتح Command Prompt في مجلد المشروع
  ج) شغّل:
       pip install -r requirements.txt
  د) شغّل البوت:
       python main.py

الطريقة الثالثة — المثبّت الرسومي (Inno Setup):
  أ) انقر نقراً مزدوجاً على NexAgent_Setup.exe
  ب) اتبع خطوات المعالج

═══════════════════════════════════
6. إعداد الطبقة الأولى — أول مرة
═══════════════════════════════════

  أ) افتح ملف nexagent_config.ini وأدخل:
       [telegram]
       bot_token   = TOKEN_الخاص_بك
       allowed_users = CHAT_ID_الخاص_بك

       [ai]
       provider    = openai    (أو gemini)
       openai_key  = sk-...    (أو gemini_key = AI...)

  ب) شغّل البوت:  python main.py
  ج) في تيليغرام ابحث عن بوتك وأرسل:  /start
  د) ستظهر قائمة الأزرار السريعة

أمثلة على أوامر يفهمها البوت:
  • "خذ لقطة شاشة"
  • "افتح المتصفح"
  • "أرسلي حالة الحاسب"
  • "اغلق الحاسب بعد 30 دقيقة"
  • "اعرض ملفات سطح المكتب"
  • "شغّل AnyDesk وأرسل الـ ID"
  • "ضبط الصوت على 50%"
  • "اقفل الشاشة"

═══════════════════════════════════
7. تثبيت الطبقة الثانية — SSH
═══════════════════════════════════

الطبقة الثانية اختيارية. استخدمها للوصول الكامل عبر SSH
(terminal، نقل ملفات، سيناريوهات غير متوفرة عبر البوت).

الخطوة 1 — الحصول على bore.exe:
  حمّل bore.exe من:
    https://github.com/ekzhang/bore/releases
  (اختر النسخة: bore-vX.X.X-x86_64-pc-windows-msvc.zip)
  ضع bore.exe في مجلد core\

الخطوة 2 — إعداد core\sshremote_config.ini:
  افتح الملف وأدخل:
    [telegram]
    bot_token = TOKEN_الخاص_بك
    chat_id   = CHAT_ID_الخاص_بك

    [install]
    install_path = C:\SshRemote_V2
    task_name    = SshRemoteTunnel_V2

    [bore]
    bore_server = bore.pub
    local_port  = 22

الخطوة 3 — تشغيل setup_v2.bat كمسؤول:
  انقر بزر الأيمن على core\setup_v2.bat
  اختر "تشغيل كمسؤول"
  انتظر حتى اكتمال التثبيت (7 خطوات تلقائية)

الخطوة 4 — تفعيل أوامر SSH في البوت:
  افتح nexagent_config.ini وأضف:
    [ssh]
    username = اسم_المستخدم_الخاص_بك

الخطوة 5 — التأكد من الاتصال:
  في تيليغرام أرسل:  /start  ثم اختر "حالة SSH"
  أو أرسل: "حالة نفق SSH"
  ستستقبل رسالة مثل:
    ✅ SshRemote متصل!
    🔗 للاتصال:
    ssh المستخدم@bore.pub -p 54321
    🔑 المفتاح: sshremote_key

═══════════════════════════════════
10. حل المشكلات الشائعة
═══════════════════════════════════

المشكلة: البوت لا يستجيب
  الحل: تأكد من صحة Bot Token — أعِد تشغيل البوت

المشكلة: "غير مصرح لك باستخدام هذا البوت"
  الحل: تأكد أن Chat ID مدرج في allowed_users

المشكلة: خطأ في مفتاح AI
  الحل: تحقق من مفتاح OpenAI أو Gemini

المشكلة: لقطة الشاشة لا تُرسل
  الحل: شغّل:  pip install Pillow pyautogui

المشكلة: البوت يتوقف بعد فترة
  الحل: فعّل watchdog_enabled = true في الإعدادات

المشكلة: أوامر vision_* لا تعمل
  الحل: هذه الأوامر تتطلب OpenAI GPT-4o فقط

المشكلة: الرسائل الصوتية لا تُحوَّل
  الحل: استخدم مزود OpenAI

المشكلة: أوامر ssh_* تعيد "SSH غير مهيأ"
  الحل: أضف username في قسم [ssh] في nexagent_config.ini

المشكلة: النفق يتوقف ولا يُرسل المنفذ
  الحل: تحقق من تشغيل bore.exe في C:\SshRemote_V2\
         شغّل core\start_tunnel_v2.bat يدوياً
         تأكد من اتصال الإنترنت

المشكلة: الاتصال بـ SSH يفشل
  الحل: تحقق من أن sshd يعمل:
         net start sshd
         تحقق من المنفذ عبر: core\show_port_v2.bat

═══════════════════════════════════
11. ملاحظات الأمان
═══════════════════════════════════

  ✓ لا تشارك ملف config.json مع أحد — يحتوي على مفاتيح مشفرة
  ✓ لا تحذف secret.key — بدونه لن تتمكن من فك تشفير إعداداتك
  ✓ فعّل رمز PIN من الإعدادات لطبقة أمان إضافية
  ✓ الطبقة الثانية تستخدم المصادقة بالمفتاح العام فقط
    (PasswordAuthentication no — بدون كلمة مرور)
  ✓ منافذ bore مؤقتة — تتغير عند كل إعادة اتصال
  ✓ لا تُضف Chat IDs لأشخاص لا تثق بهم تماماً

═══════════════════════════════════
12. أمان مفاتيح SSH
═══════════════════════════════════

  ⚠️  لا تلتزم مطلقاً بملفات مفاتيح SSH في Git!
      ملفا sshremote_key و sshremote_key.pub مُدرجان في .gitignore —
      لا تحذف هذه القاعدة أبداً.

  توليد زوج مفاتيح خاص بك:
    1. افتح Terminal أو PowerShell
    2. شغّل الأمر:
         ssh-keygen -t ed25519 -f sshremote_key
    3. سيُنشئ الأمر ملفَّين:
         sshremote_key       ← المفتاح الخاص  (احتفظ به سرياً!)
         sshremote_key.pub   ← المفتاح العام
    4. انسخ محتوى sshremote_key.pub إلى ملف authorized_keys
       على الجهاز الهدف (عادةً في ~/.ssh/authorized_keys)
    5. ضع ملف sshremote_key في مجلد core\ فقط على جهازك
       ولا ترفعه إلى أي مستودع أو خدمة سحابية
