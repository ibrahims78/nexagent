1. WHAT IS NEXAGENT?
═══════════════════════════════════

NexAgent is a two-layer Windows PC remote-control system:

  Layer 1 — AI Telegram Bot:
    Control your Windows PC with natural language commands (Arabic or English).
    Powered by OpenAI GPT-4o or Google Gemini 1.5 Flash.

  Layer 2 — SSH Tunnel (SshRemote V2):
    Passwordless SSH access through an automatic bore tunnel.
    Works without a static IP or router port-forwarding.
    Sends the new connection port to Telegram on every reconnect.

Key capabilities:
  - Instant screenshot
  - Open / close applications
  - File management
  - CPU / RAM / temperature monitoring
  - AI vision control (sees the screen and clicks)
  - Wake-on-LAN (remotely power on your PC)
  - Live screen streaming
  - Scheduled daily reports
  - Shell commands via SSH
  - File upload/download via SFTP

═══════════════════════════════════
2. SYSTEM REQUIREMENTS
═══════════════════════════════════

  - Windows 10 or 11 (64-bit)
  - Python 3.11 or newer  <  python.org/downloads  >
    (during install: check "Add Python to PATH")
  - Internet connection
  - API key: OpenAI (gpt-4o) or Google Gemini
  - For Layer 2: Administrator rights to install OpenSSH

═══════════════════════════════════
3. CREATE A TELEGRAM BOT
═══════════════════════════════════

  a) Open Telegram and search for @BotFather
  b) Send the command:  /newbot
  c) Choose a display name (e.g. My NexAgent Bot)
  d) Choose a username ending in "bot" (e.g. my_nexagent_bot)
  e) Save the "HTTP API token" BotFather gives you
     Example: 7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

═══════════════════════════════════
4. FIND YOUR CHAT ID
═══════════════════════════════════

  a) Open Telegram and search for @userinfobot
  b) Send any message to it
  c) It will reply with your info — save the "Id" number
     Example: 123456789

═══════════════════════════════════
5. INSTALL LAYER 1 — AI BOT
═══════════════════════════════════

Method 1 — Automatic installer (recommended):
  a) Right-click install.bat
  b) Choose "Run as administrator"
  c) Follow the on-screen instructions

Method 2 — Manual setup:
  a) Make sure Python 3.11+ is installed
  b) Open Command Prompt in the project folder
  c) Run:
       pip install -r requirements.txt
  d) Start the bot:
       python main.py

Method 3 — GUI installer (Inno Setup):
  a) Double-click NexAgent_Setup.exe
  b) Follow the wizard steps

═══════════════════════════════════
6. CONFIGURE LAYER 1 — FIRST TIME
═══════════════════════════════════

  a) Open nexagent_config.ini and fill in:
       [telegram]
       bot_token   = YOUR_BOT_TOKEN
       allowed_users = YOUR_CHAT_ID

       [ai]
       provider    = openai    (or gemini)
       openai_key  = sk-...    (or gemini_key = AI...)

  b) Start the bot:  python main.py
  c) Find your bot on Telegram and send:  /start
  d) A quick-command keyboard will appear

Example commands the bot understands:
  - "Take a screenshot"
  - "Open Chrome"
  - "What's the PC status?"
  - "Shut down in 30 minutes"
  - "List Desktop files"
  - "Start AnyDesk and send me the ID"
  - "Set volume to 50%"
  - "Lock the screen"

═══════════════════════════════════
7. INSTALL LAYER 2 — SSH TUNNEL
═══════════════════════════════════

Layer 2 is optional. Use it for full SSH terminal access and SFTP
(scenarios that go beyond what the bot commands cover).

Step 1 — Download bore.exe:
  Get it from:
    https://github.com/ekzhang/bore/releases
  (choose: bore-vX.X.X-x86_64-pc-windows-msvc.zip)
  Place bore.exe in the core\ folder.

Step 2 — Edit core\sshremote_config.ini:
  Fill in:
    [telegram]
    bot_token = YOUR_BOT_TOKEN
    chat_id   = YOUR_CHAT_ID

    [install]
    install_path = C:\SshRemote_V2
    task_name    = SshRemoteTunnel_V2

    [bore]
    bore_server = bore.pub
    local_port  = 22

Step 3 — Run setup_v2.bat as Administrator:
  Right-click core\setup_v2.bat
  Choose "Run as administrator"
  Wait for all 7 steps to complete

Step 4 — Enable SSH commands in the bot:
  Open nexagent_config.ini and add:
    [ssh]
    username = your_windows_username

Step 5 — Verify the connection:
  In Telegram send: "SSH status"
  You should receive:
    ✅ SshRemote connected!
    🖥️ Device: DESKTOP-ABC
    🔗 To connect:
    ssh user@bore.pub -p 54321
    🔑 Key: sshremote_key

8. SSH COMMANDS VIA THE BOT
═══════════════════════════════════

  Command              Description
  ─────────────────────────────────────────────────────
  "SSH status"         Show tunnel status + connection string
  "Run via SSH dir C:\" Execute a shell command via SSH
  "List SSH files"     Browse remote directory via SFTP
  "Download report.pdf" Download file from PC via SFTP
  "Current tunnel port" Show active bore port number

10. TROUBLESHOOTING
═══════════════════════════════════

Problem: Bot does not respond
  Fix: Verify the Bot Token — restart the bot

Problem: "You are not authorized to use this bot"
  Fix: Make sure your Chat ID is in the allowed_users field

Problem: AI key error
  Fix: Check your OpenAI or Gemini key

Problem: Screenshots not sending
  Fix: Run:  pip install Pillow pyautogui

Problem: Bot stops after a while
  Fix: Set watchdog_enabled = true in settings

Problem: vision_* commands not working
  Fix: These require OpenAI GPT-4o only — not available with Gemini

Problem: Voice messages not transcribed
  Fix: Use the OpenAI provider

Problem: ssh_* commands return "SSH not configured"
  Fix: Add ssh.username to the [ssh] section of nexagent_config.ini

Problem: Tunnel stops / no port received
  Fix: Check bore.exe is in C:\SshRemote_V2\
       Run core\start_tunnel_v2.bat manually
       Verify internet connection

Problem: SSH connection fails
  Fix: Confirm sshd is running:
         net start sshd
       Check the port via: core\show_port_v2.bat

Problem: Build error "Python not found"
  Fix: Install Python and check "Add Python to PATH" during setup

═══════════════════════════════════
11. SECURITY NOTES
═══════════════════════════════════

  - Never share config.json — it contains encrypted keys
  - Never delete secret.key — without it you cannot decrypt your settings
  - Enable PIN in settings for an extra security layer
  - Layer 2 uses public-key auth only (PasswordAuthentication no — no passwords)
  - bore ports are ephemeral — they change on every reconnect
  - Only add Chat IDs of people you fully trust — they get complete PC control
